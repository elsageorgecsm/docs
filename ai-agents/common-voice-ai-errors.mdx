---
title: "Common Voice AI Errors"
---

This page documents common errors encountered in Voice AI systems.\
Voice AI pipelines consist of multiple components, and failures can occur at any layer.

Typical pipeline:

Audio Capture → VAD → Streaming → STT → Semantic Processing → LLM → Tools → TTS → WebRTC → Network → Output

Understanding the failing layer helps quickly isolate issues.

---

# Audio Capture Errors

### Microphone Permission Denied

**Reason**

- Browser microphone permission not granted
- OS-level microphone access disabled

**Example**

```text
DOMException: Permission denied
```

**Fix**

- Enable microphone permissions in browser settings
- Verify OS-level microphone access

---

### No Audio Input Detected

**Reason**

- Microphone not connected
- Incorrect input device selected

**Example**

```text
No audio frames received
```

**Fix**

- Select correct microphone
- Test microphone input locally

---

### Unsupported Audio Format

**Reason**

- Unsupported codec
- Invalid sample rate

**Example**

```text
Unsupported audio format
```

**Fix**

Convert audio to supported format such as:

```text
PCM
16kHz mono
WAV
```

---

### Audio Sample Rate Mismatch

**Reason**

STT model requires specific sample rate.

**Example**

```text
Invalid sample rate: expected 16000
```

**Fix**

Resample audio before sending.

---

### Audio Device Busy

**Reason**

Another application is using the microphone.

**Example**

```text
Audio device already in use
```

**Fix**

Close other applications using the microphone.

---

# Voice Activity Detection (VAD) Errors

### Speech Not Detected

**Reason**

- Low microphone gain
- VAD sensitivity too high

**Example**

```text
VAD: no speech detected
```

**Fix**

Adjust VAD threshold.

---

### False Speech Trigger

**Reason**

- Background noise
- Echo

**Example**

```text
VAD triggered by noise
```

**Fix**

Enable noise suppression.

---

### Early Speech Cutoff

**Reason**

Silence detection triggered too early.

**Example**

```text
Speech segment ended prematurely
```

**Fix**

Increase silence timeout.

---

### Continuous Speech Detected

**Reason**

Noise causing VAD to remain active.

**Example**

```text
Speech segment never closed
```

**Fix**

Tune VAD parameters.

---

# Streaming Errors

### Audio Stream Interrupted

**Reason**

- Network interruption
- Client disconnect

**Example**

```text
Audio stream interrupted
```

**Fix**

Reconnect stream.

---

### Streaming Buffer Overflow

**Reason**

Audio frames arriving faster than processing.

**Example**

```text
Audio buffer overflow
```

**Fix**

Increase buffer size.

---

### Streaming Backpressure

**Reason**

Downstream services slower than input.

**Example**

```text
Stream backpressure detected
```

**Fix**

Throttle input stream.

---

### Stream Timeout

**Reason**

No audio frames received.

**Example**

```text
Stream timeout
```

**Fix**

Send keepalive packets.

---

```

**Reason**

```

**Reason**

STT model cannot decode format.

**Example**

```text

```

**Fix**

Convert audio before sending.

---

**Reason**

**Example**

```text

```

---

### STT Confidence Too Low

**Reason**

Speech unclear or noisy.

**Example**

```text
Low transcription confidence
```

**Fix**

Improve audio quality.

# Debugging Workflow

When debugging Voice AI systems:

1. Identify the failing pipeline stage.
2. Check logs for errors and request IDs.
3. Validate audio format and streaming configuration.
4. Monitor latency, packet loss, and jitter.
5. Implement retries and reconnection logic.

Breaking debugging down by pipeline layer dramatically simplifies troubleshooting.
