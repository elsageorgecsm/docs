## Overview

The API reference section provides detailed information about the core components used to build voice agents.

These APIs allow developers to manage:

- agent sessions
- server runtime
- plugins and integrations
- communication with LiveKit rooms

This reference section complements the guides by explaining **how the underlying components work and how to use them in code**.

---

## Key Components

The main components covered in this reference include:

### AgentSession

The `AgentSession` manages the lifecycle of a voice agent during a conversation.

Responsibilities include:

- handling audio streams
- coordinating speech-to-text
- sending messages to the language model
- generating responses

---

### AgentServer

The `AgentServer` runs the voice agent service and manages connections to the LiveKit server.

It handles:

- incoming sessions
- agent lifecycle management
- communication with clients

---

### Plugins

Plugins extend the functionality of voice agents.

They may include integrations for:

- speech-to-text engines
- text-to-speech systems
- noise cancellation
- external APIs

Plugins allow developers to customize how their voice agents process audio and generate responses.

---

## Example Structure

A simple voice agent setup may look like this:

```python
from livekit.agents import AgentSession, AgentServer

session = AgentSession(
    stt="speech_to_text_engine",
    llm="language_model",
    tts="text_to_speech_engine"
)

server = AgentServer()
server.start()
```

In this example:

- `AgentSession` manages the conversation
- `AgentServer` runs the agent service
